# crypto-slang-tone

## На вход

- `workspace/news.json` (результат `fetch-crypto-news`).
- По умолчанию берёт первый элемент (топ-1).

## Требования к твиту

- Длина: **180–240 символов**.
- Тон: ироничный, слегка саркастичный.
- Крипто-сленг: gm, ngmi, wagmi, rekt, ser, anon, bullish, bearish,
  degen, aped in, copium, hopium, probably nothing, few understand,
  bagholder.
- Тикеры с долларом: `$BTC`, `$ETH`, `$SOL`.
- Максимум **1 эмодзи**, лучше без.
- Максимум **1 хэштег** и только если он реально уместен.
- Никаких `BREAKING`, никакого КАПСА.

## На выход

Пишет в stdout финальный текст твита (чистая строка, без кавычек).
Скрипт также сохраняет промежуточный результат в `workspace/tweet_raw.txt`.

## Запуск

```bash
python .claude/skills/crypto-slang-tone/rewrite.py
```

## Логика

Скрипт формирует каркас твита из заголовка и summary новости,
добавляет $ к тикерам, вставляет уместный сленговый оборот
(выбор зависит от типа события: hack → rekt/ngmi, pump → wagmi/bullish,
launch → probably nothing/few understand), обрезает до 240 символов.
Доводку тона делает человек (или LLM) через этот файл — тут
детерминированный базовый черновик.
